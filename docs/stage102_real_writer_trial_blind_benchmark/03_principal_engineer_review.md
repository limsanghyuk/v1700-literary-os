# Stage102 Principal Engineer Review

## Judgment

Stage102 is valid if it stays evidence-focused. It must not pretend to be a
large-scale market study or a real external human reader panel.

## Required Controls

- Run mandatory pre-development protocol before implementation.
- Keep GitNexus as optional sidecar.
- Preserve GraphNexus and Branchpoint authority.
- Keep provider calls at zero in release mode.
- Keep raw manuscript text inside local evidence only.
- Keep Node2 surface-only reveal boundary.
- Keep direct V430 runtime merge forbidden.

## Problem Checks

| Problem | Risk | Resolution |
|---|---|---|
| Internal fixture benchmark overclaim | False product confidence | Mark benchmark as local deterministic evidence |
| Direct provider-style baselines too weak | Inflated V1700 margin | Keep multiple baselines: GPT, Claude, Gemini, Ollama |
| Scenario-room absorption contaminates prose scoring | Evaluation confusion | Preserve Stage100 dual-mode separation |
| Writer trial becomes UI scope | Stage bloat | Keep tasks as evidence contracts, not UI implementation |
| Missing branchpoint trace | Orphan logic | Add Stage102 symbol trace entries |

## Approval

Approved as a controlled Stage102 implementation before any larger real-user or
product beta stage.
