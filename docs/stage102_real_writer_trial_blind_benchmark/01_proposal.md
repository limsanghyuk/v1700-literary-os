# Stage102 Proposal

## Title

V1700 Stage102 - Real Writer Trial & Blind Benchmark

## Purpose

Stage102 validates whether the Stage101 engine is useful in a writer-facing
workflow. The goal is not to add another large generation engine. The goal is to
prove that the existing V1700 logic can be evaluated through repeatable writer
tasks, blind candidate scoring, and revision-efficiency evidence.

## Senior Architect Position

Stage101 created a scenario room and cross-lineage absorption layer. Stage102
must test that layer in a controlled writer workflow:

- same seed prompt
- multiple candidate modes
- blinded scorecards
- writer task friction
- revision time reduction
- plot, payoff, and scene necessity audit

The architect rejects broad UI expansion at this stage. Product UI belongs to a
later productization pass.

## Senior Compiler Position

Stage102 should compile into small contracts and gates:

- seed bank
- trial candidates
- blind scorecards
- writer task results
- revision efficiency report
- release evidence pack

The compiler requires all evidence to be machine-readable and generated without
live provider calls.

## System Principal Engineer Position

The main risk is overclaiming. Stage102 must clearly state that its benchmark is
deterministic internal evidence, not an external human market study.

Blocking risks:

- blind identity leakage
- V1700 not beating direct baseline
- provider call count above zero
- raw manuscript leakage
- Node2 raw reveal access
- missing branchpoint trace

## Final Proposal

Proceed with Stage102 as a local-first, deterministic writer trial and blind
benchmark stage. Use it as the bridge from internal architecture proof toward
future real user trials.
