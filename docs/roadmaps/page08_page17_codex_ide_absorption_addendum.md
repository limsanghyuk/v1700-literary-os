## Gate policy absorbed from the proposal

Blocking gates are limited to safety, continuity, authority, and integrity. Advisory gates express literary diagnosis.

```text
Blocking examples:
- Provider policy violation
- write before approval
- Node2 boundary violation
- canon consistency failure
- character knowledge boundary failure
- forbidden reveal failure
- sandbox isolation failure
- merge approval failure
- metadata consistency failure
- release asset integrity failure

Advisory examples:
- EAT8D literary signal
- reader surface note
- dialogue naturalness note
- microplot density note
- agency balance note
- emotion curve note
- plot density warning
```

## Acceptance criteria absorbed from the proposal

The Codex-style Narrative Agent Core is not ready until:

```text
1. EAT8D features distinguish required, optional, and missing fields.
2. Scene or render packet inputs produce traceable features.
3. 8D values include confidence and provenance.
4. StoryPatch records affected character, episode, and payoff units.
5. Narrative Gate Runner separates blocking and advisory results.
6. All patch application occurs inside a sandbox until approval.
7. Main story state cannot be changed before writer approval.
8. Narrative PR includes diff, gate results, and review checklist.
9. Release asset integrity uses standard verification.
10. Release gates do not rely on stale pass artifacts.
```
