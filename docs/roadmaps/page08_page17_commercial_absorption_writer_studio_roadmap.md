## Development principle

Commercial writing features must not be copied directly. They must be translated into V1700 form:

```text
commercial feature -> contract -> evidence schema -> fixture pack -> boundary gate -> dry-run -> author approval where needed -> signed release
```

The roadmap therefore starts with lineage survival rather than new user-facing features.

## Inherited invariants

All work in Page08~Page17 must preserve these inherited rules unless a later stage formally deprecates them with successor trace:

```text
Provider-Zero remains the default.
Node2 remains surface-safe.
Memory writes remain disabled until a write contract exists.
Canon mutation remains disabled until an approved mutation path exists.
Runtime learning remains disabled until audit and rollback exist.
ASD remains non-mutating until patch proposal, author approval, and reversible commit exist.
8D literary values remain advisory.
GIG / contradiction intelligence begins advisory.
Project isolation precedes MultiWork sharing.
License boundary precedes cross-project access.
```
